Oui l'impression sous windows produit un texte trop petit

Ah, un problème classique avec GTK sous Windows ! L'impression GTK peut avoir des soucis de scaling/DPI sous Windows. Voici quelques pistes pour corriger ça dans votre code C :

Solutions possibles
1. Ajuster le scaling du contexte Cairo
Dans votre fonction d'impression, vous pouvez forcer un scale plus grand pour Windows :

c
static void draw_page(GtkPrintOperation *operation,
                     GtkPrintContext *context,
                     gint page_nr,
                     gpointer user_data) {
    cairo_t *cr = gtk_print_context_get_cairo_context(context);
    
    #ifdef _WIN32
    // Augmenter le scale sous Windows
    cairo_scale(cr, 1.5, 1.5);  // Ajuster le facteur selon les besoins
    #endif
    
    // Reste du code de dessin...
}
2. Ajuster la taille de police spécifiquement pour Windows
c
static void set_font_size(cairo_t *cr, double size) {
    #ifdef _WIN32
    cairo_set_font_size(cr, size * 1.5);  // Multiplier par un facteur
    #else
    cairo_set_font_size(cr, size);
    #endif
}
3. Utiliser les DPI du contexte d'impression
c
static void draw_page(GtkPrintOperation *operation,
                     GtkPrintContext *context,
                     gint page_nr,
                     gpointer user_data) {
    cairo_t *cr = gtk_print_context_get_cairo_context(context);
    
    // Récupérer les DPI
    gdouble dpi_x = gtk_print_context_get_dpi_x(context);
    gdouble dpi_y = gtk_print_context_get_dpi_y(context);
    
    #ifdef _WIN32
    // Windows rapporte souvent des DPI incorrects, forcer à 96 ou 120
    if (dpi_x < 96.0) {
        gdouble scale = 96.0 / dpi_x;
        cairo_scale(cr, scale, scale);
    }
    #endif
    
    // Suite du dessin...
}
4. Solution via Pango (si vous utilisez Pango pour le texte)
c
PangoFontDescription *font_desc;
font_desc = pango_font_description_new();

#ifdef _WIN32
pango_font_description_set_size(font_desc, 12 * PANGO_SCALE * 1.5);
#else  
pango_font_description_set_size(font_desc, 12 * PANGO_SCALE);
#endif
5. Variable d'environnement GDK
Vous pouvez aussi documenter dans le README pour les utilisateurs Windows :

6. Option de configuration utilisateur
Ajouter une préférence dans l'interface pour ajuster le scale d'impression :

c
// Dans la structure de config
typedef struct {
    double print_scale;  // 1.0 par défaut, 1.5 pour Windows
    // ...
} AppConfig;

// Dans les préférences (ou détection auto)
#ifdef _WIN32
config.print_scale = 1.5;
#else
config.print_scale = 1.0;
#endif

// À l'impression
cairo_scale(cr, config.print_scale, config.print_scale);
